# 井字棋获胜判断 —— 矩阵模式匹配法

## 思路来源：寻找不变量

井字棋的规则决定了一个不变的事实：

> **无论棋局如何演变，获胜的形态永远只有 8 种——3 行、3 列、2 条对角线。**

既然不变量是固定的，那与其在棋盘上逐个搜索有没有三连，不如**把 8 种获胜形态预先编码成模式矩阵**，然后用测试棋盘与它们逐一做差。差值 0 和 -1 就是胜者的唯一指纹。

## 编码设计

我们需要一种编码方式，使得做差结果能直接告诉我们谁赢了。

| 对象 | 编码 | 含义 |
|------|:----:|------|
| A 的棋子（X） | **1** | 占据位置 |
| B 的棋子（O） | **0** | 占据位置 |
| 空位 | **2** | 未被占据 |
| 模式矩阵中的胜位 | **1** | 需要匹配的关键位置 |
| 模式矩阵中的无关位 | **5** | 不关心的位置 |

做差的几种情况：

```
A 占胜位：  1 - 1 =  0  ✅ 匹配
B 占胜位：  0 - 1 = -1  ✅ 匹配
空位无关位： 2 - 5 = -3  ❌ 不干扰
A 占无关位： 1 - 5 = -4  ❌ 不干扰
B 占无关位： 0 - 5 = -5  ❌ 不干扰
```

0 和 -1 是所有差值中**唯二**对应胜位匹配的情况，保证了判断的绝对唯一性。

## 代码

```python
class Solution:
    def tictactoe(self, moves):
        board = [[2]*3 for _ in range(3)]
        for i, (r, c) in enumerate(moves):
            board[r][c] = 1 if i % 2 == 0 else 0

        patterns = [
            [[1,1,1],[5,5,5],[5,5,5]],
            [[5,5,5],[1,1,1],[5,5,5]],
            [[5,5,5],[5,5,5],[1,1,1]],
            [[1,5,5],[1,5,5],[1,5,5]],
            [[5,1,5],[5,1,5],[5,1,5]],
            [[5,5,1],[5,5,1],[5,5,1]],
            [[1,5,5],[5,1,5],[5,5,1]],
            [[5,5,1],[5,1,5],[1,5,5]],
        ]

        for pat in patterns:
            diff = [[board[i][j] - pat[i][j] for j in range(3)] for i in range(3)]
            zeros = sum(row.count(0) for row in diff)
            neg_ones = sum(row.count(-1) for row in diff)
            
            if zeros == 3:
                return "A"
            if neg_ones == 3:
                return "B"

        has_empty = any(2 in row for row in board)
        return "Pending" if has_empty else "Draw"
```

## 其他解法对比

| 解法 | 思路 | 特点 |
|------|------|------|
| 坐标匹配法 | 用集合存 A、B 的落子位置，检查是否包含某获胜三元组 | 直观，但需要多次 in 查找 |
| 行/列/对角线求和 | 3×3 整数数组，A=1, B=-1，检查和是否为 3 或 -3 | 简洁，需要索引映射 |
| **矩阵模式匹配法（本文）** | 预编码 8 种获胜模式，做差匹配 | 模板化思维，数值判断，和而不同 |

---

如果把棋盘比作锁，那 8 种获胜形态就是锁芯里固定的弹子排列。每一局棋都是不同的钥匙插入，匹配上了，锁就开了。
